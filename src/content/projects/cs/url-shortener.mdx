## Kontext

Zkracovač URL je proslule jednoduchý na postavení – a proslule těžký na postavení *pořádně*
ve velkém. Použil jsem ho jako záminku k procvičení věcí, které monolit schovává:
hranice služeb, strategie cachování, asynchronní pipeline a observabilita.

## Problém

Zajímavé problémy se objeví, až když odmítnete snadnou cestu: co se stane při bouři cache
missů? Jak zaznamenat kliknutí, aniž by zpomalilo přesměrování? Jak poznáte, že analytická
pipeline potichu spadla – a jak se systém zachová, když se to stane?

## Řešení

Tři služby a gateway, každá s přísným kontraktem:

- **Shortener služba** – zápisová cesta: validace, generování base62 kódů odolné proti
  kolizím, PostgreSQL úložiště
- **Redirector služba** – horká čtecí cesta: Redis cache-first s Postgres fallbackem;
  každý hit odpálí click event do **Redis Streamu**, aniž by kdy blokoval přesměrování
- **Analytics služba** – consumer-group čtečka agreguje stream do denních rollupů;
  stats API servíruje součty, trendy a top referrery
- **API gateway** – jediný veřejný povrch: rate limiting, propagace request-id
  a overview endpoint s **graceful degradation** (informace o URL odpoví, i když
  analytika leží)

Celý stack běží z jednoho `docker compose up` – Postgres, Redis, všechny čtyři služby,
Prometheus scrapující každý `/metrics` a provisionovaný Grafana dashboard s request rate
a p95 latencí. Kubernetes manifesty jsou přiložené jako referenční konfigurace. Každá
služba loguje strukturovaný JSON, má skutečné `/health` checky a umí graceful shutdown.

## Výsledek

- Přesměrování s cache hitem se nikdy nedotkne databáze; analytika se nikdy nedotkne horké cesty
- Gateway odpovídá dál – degradovaně, označeně, ale běží – když downstream služba umře
- **56 testů běží úplně bez infrastruktury**: každá služba exportuje `createApp(deps)`
  a testy injektují in-memory fakes
- Golden-signal dashboardy rovnou v základu

## Co jsem se naučil

Observabilita je feature, kterou stavíte *jako první*, ne poslední – a druhé nejlepší
rozhodnutí byla dependency injection všude: systém, který otestujete na notebooku
bez jediné běžící závislosti, je systém, kterému skutečně rozumíte.
